# Sleep Thera 랜딩페이지 1-Pager

> **Sleep Thera**는 벨 테라퓨틱스가 개발한 뇌과학 기반 불면증 디지털 치료제. 이 문서는 초기 사용자 모집 전략을 다룬다.

## 프로젝트 개요
**초기 사용자 모집을 위한 예비 사용자 풀 구축**

---

## 배경

| 현황 | 문제 |
|------|------|
| 앱이 외부에 공개되지 않음 | 실제 사용자가 없음 |
| 현재 사용자는 임상 참여자뿐 | 임상 중에는 직접 유저 리서치 진행 어려움 |
| 허가 후 시장 출시 대비 필요 | 출시 시 좋은 반응을 위한 사전 검증 필요 |
| 신사업/신규 기능 검증 필요 | 빠른 이터레이션을 위한 유저 풀 부재 |

**→ 허가 전부터 예비 사용자 풀을 구축하여, 지속적인 이터레이션을 통해 출시 시 제품 완성도를 높인다**

## 타겟

**ISI 15점 이상 + 수면제 부작용 경험자**

| 조건 | 설명 |
|------|------|
| **ISI 15점 이상** | 중등도~심각 불면증 (약을 먹을 정도로 심각한 수준) |
| **수면제 부작용 경험** | 약을 먹어봤지만 다음 날 힘들었던 사람들 |

> **ISI 15점 기준 근거**: 확증임상에서 ISI 15점 이상 환자를 대상으로 진행 중이며, 허가 후 제공 역시 동일한 대상에게 이루어질 예정.

### 왜 이 타겟인가

우리 USP = "효과도 있고(검증됨) + 부작용도 없다"

| 옵션 | 효과 | 부작용 |
|------|------|--------|
| 수면제 | O | O |
| 일반 수면 앱 | ? | X |
| **디지털 치료제** | **O** | **X** |

이 USP에 가장 공감할 세그먼트 = 약 부작용을 직접 겪어본 사람
- 약을 먹어봤다 → 불면증이 심각하다 (ISI 15점+ 가능성 높음)
- 부작용을 겪었다 → 대안을 찾고 있다 (전환 의지 높음)

### 근거 (UT3 유저 인터뷰)

**1. 약물 부작용으로 인한 고통**
> "부작용이 엄청 힘들었음. 속이 넘 메스꺼워서... 수면제는 안 맞으면 찾는 과정이 있는데 그게 몸이 예민해지고 머리 아프고 그런게 있음"

**2. 앱은 부작용이 없다는 장점**
> "약물 복용 시랑 앱과의 효과를 비교했을 때 앱이 낫다. 앱은 써서 잘 못 자도 몸이 변하지 않아서 (부작용 없어서) 좋았음"

**3. 수면제 감소로 인한 부가 효과**
> "수면제가 우울증에 영향을 줬는데, 수면제 복용이 줄게 되어 우울증이 좀 더 개선된 느낌"

**4. 약보다 빠른 입면 효과**
> "본인 입면이 약을 먹어도 1~2시간이 걸리는데 그 원인이 핸드폰을 봐서라는 것을 이 앱을 통해서 깨닫게 됨. 그 이후 30분 안에 잠을 들지 못한 적이 없음"

## 모집 방법

### 전략 배경

- 첫 웹사이트, Organic 검색 의존 → 트래픽이 적을 것
- 적은 트래픽에서 5명 등록하려면 전환율이 높아야 함
- 뾰족한 메시지로 특정 세그먼트의 전환율을 높이는 전략

### 전략 1: 타겟 메시지 + SEO 틈새

**접근**
- "불면증" 같은 넓은 키워드 대신 "수면제 부작용 대안" 같은 틈새 키워드로 노출

**가정**
- 뾰족한 메시지가 전환율을 높인다
- 틈새 키워드가 검색에서 노출된다
- 수면제 부작용 경험자가 일반 불면증 환자보다 전환율이 높다

**검증**
- 등록자 중 부작용 경험 비율 확인 (40%+ 목표)
- Google Search Console - 노출/클릭 키워드

### 전략 2: 테스트 바이럴

**접근**
- 수면 테스트 결과 공유 유도
- OG 이미지: "내 수면 건강은 하위 X%"
- 결과 페이지에서 공유 버튼 제공

**가정**
- 테스트 결과를 사람들이 공유하고 싶어한다
- "하위 X%" 형태가 공유 욕구를 자극한다

**검증**
- 공유 버튼 클릭 수
- Referral 트래픽 비율

### 트래픽 채널

| 채널 | 설명 |
|------|------|
| 벨 테라퓨틱스 홈페이지 | 기존 회사 홈페이지에서 랜딩페이지로 유입 경로 추가 |
| Organic 검색 | 기존 웹사이트 insomnia 섹션 방문자 유입 (월 53명 기준) |
| 지인/팀 공유 | 초기 트래픽 확보를 위한 내부 공유 |

**가정**: Organic + 기존 웹사이트로 충분한 초기 트래픽 확보
**검증**: GA 채널별 유입량 (월 53명 기준 대비)

## 검증 방법: 수면 테스트 페이지

**목적**: 타겟 유저가 본인의 적합도를 스스로 인지하도록 유도

```
"내 불면증은 얼마나 심각할까?" 테스트
   ↓
Part 1: 수면 고민 선택
   ↓
Part 2: ISI 7문항 + 조건부 후속 질문
   ↓
[ISI 8점+ only] Part 3: 추가 정보 (지속기간, 수면제, 성별, 연령대)
   ↓
결과: 점수 + 심각도 등급
   ↓
결과 공유 유도 (바이럴)
   ↓
Waiting List 등록 CTA
```

| 점수 | 등급 | 메시지 |
|------|------|--------|
| 0-7 | 정상 | "수면 건강이 양호하시네요" |
| 8-14 | 경미 | "가벼운 불면 증상이 있어요. 지금부터 관리하면 좋아요" |
| 15-21 | 중등도 | "많은 분들이 비슷한 상황에서 시작했어요. 함께 개선해봐요" |
| 22-28 | 심각 | "쉽지 않은 밤을 보내셨겠어요. 저희가 도움이 될 수 있어요" |

> 15점 미만도 등록은 가능하되, DB에서 점수로 구분하여 서비스 제공 우선순위 결정
