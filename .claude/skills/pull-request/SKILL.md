# 명세 변경 PR 생성

현재 브랜치의 커밋들을 분석하여 구조화된 PR을 생성한다.

## 언어

PR 제목과 본문은 **모두 한글**로 작성한다.

## 실행 절차

### 1. 커밋 분석

현재 브랜치의 커밋에서 `[SPEC]` 라인이 있는 커밋만 수집하여 변경된 기능 코드 목록을 확정한다. `[SPEC]` 라인이 없는 커밋(docs, chore 등)은 테이블에 포함하지 않는다.

```bash
git log main..HEAD --format="%B" | grep "^\[SPEC\]"
```

### 2. PR 생성

#### PR 제목

```
[Spec] <project-name>: <변경 요약>
```

#### PR 본문

```markdown
## 프로젝트명

[프로젝트 이름]

## 변경 요약

[1-2문장으로 전체 변경 요약]

## 변경된 사항

> `[SPEC]`/`[TEAM]`이 있는 명세 변경 커밋만 나열한다. 기타 작업 커밋(docs, chore 등)은 포함하지 않는다. 각 항목의 설명은 커밋 body를 그대로 옮겨 적는다.

### CODE-NNN

- 페이지: page
- 변경유형: feat/update/remove
- 팀: Frontend

[커밋 body 내용]

## 참고사항

[디자이너/개발자가 알아야 할 컨텍스트]

## 관련 문서

- [변경된 페이지 문서](링크)
```

## 예시

### PR 제목

```
[Spec] update-register-flow: 등록 플로우 개선
```

### PR 본문

```markdown
## 프로젝트명

update-register-flow

## 변경 요약

등록 폼에 휴대폰번호 필드를 추가하고, 등록 완료 후 테스트 연결 메시지를 개선

## 변경된 사항

### REG-002-b

- 페이지: register
- 변경유형: feat
- 팀: Frontend

콘텐츠:
| ID | 요소 | 내용 | 인터랙션 |
|----|------|------|----------|
| REG-002-b | 휴대폰번호 입력 | 선택 필드 (미입력 가능) | 입력 시 형식 검증 |

테스트 시나리오:
- 휴대폰번호 입력 → 형식 검증 통과 → 등록 가능
- 휴대폰번호 미입력 → 등록 가능 (선택 필드)
- 잘못된 형식 입력 → 오류 메시지 표시

### REG-002-b

- 페이지: register
- 변경유형: feat
- 팀: Backend

콘텐츠:
| ID | 요소 | 내용 | 인터랙션 |
|----|------|------|----------|
| REG-002-b | 휴대폰번호 입력 | 선택 필드 (미입력 가능) | 입력 시 형식 검증 |

데이터 처리:
- 휴대폰번호 필드 추가 (nullable)
- 등록 API에서 휴대폰번호 수신 및 저장

### REG-004

- 페이지: register
- 변경유형: update
- 팀: Frontend

Before:
| ID | 요소 | 내용 | 인터랙션 |
|----|------|------|----------|
| REG-004 | 완료 메시지 | 등록 완료 안내 | - |

After:
| ID | 요소 | 내용 | 인터랙션 |
|----|------|------|----------|
| REG-004 | 완료 메시지 | 등록 완료 + 테스트 연결 안내 | 테스트 시작 CTA 클릭 → 테스트 페이지 이동 |

### LAND-009-b

- 페이지: landing
- 변경유형: update
- 팀: Frontend

Before:
| ID | 요소 | 내용 | 인터랙션 |
|----|------|------|----------|
| LAND-009-b | 폼 | 연락처 + 동의 (이메일, 동의) | 입력 → 버튼 활성화 |

After:
| ID | 요소 | 내용 | 인터랙션 |
|----|------|------|----------|
| LAND-009-b | 폼 | 연락처 + 동의 (이메일, 휴대폰번호, 동의) | 입력 → 버튼 활성화 |

## 참고사항

- 휴대폰번호는 선택 필드이므로 미입력 시에도 등록 가능해야 함
- LAND-009-b는 register.md의 REG-002와 동일한 폼

## 관련 문서

- [register.md 변경점](docs/pages/register.md)
- [landing.md 변경점](docs/pages/landing.md)
```

## 생성 전 체크리스트

- 모든 `[SPEC]` 커밋이 변경된 사항에 나열됨
- 각 항목의 설명이 커밋 body와 일치함
