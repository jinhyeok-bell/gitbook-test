# 기능 명세 작성 스킬

## 용도
`docs/pages/` 폴더 내 페이지별 기능 명세 문서 작성 시 사용

## 템플릿 구조

```markdown
# [페이지명] (/경로)

> **전체 구조**: [OVERVIEW.md](../OVERVIEW.md) 참조

---

## 1. 페이지 목적

[한 줄로 페이지의 핵심 목적 기술]

---

## 2. 사용자 플로우

[플로우 다이어그램 또는 단계별 설명]

---

## 3. 기능 명세

### [CODE-001] 기능명

**주요 기능**
[기능 설명]

**레이아웃**
| 요소 | Desktop | Mobile |
|------|---------|--------|
| ... | ... | ... |

**프로토타입**: [링크]

**테스트 케이스**
- [동작 1]
  - 성공: [기대 결과]
- [동작 2]
  - 성공: [기대 결과]

---

### [CODE-002] 기능명
...
```

## 작성 규칙

### 기능 코드 네이밍
| 페이지 | Prefix |
|--------|--------|
| 랜딩페이지 | LAND- |
| 테스트 페이지 | TEST- |
| 등록 페이지 | REG- |

### 레이아웃 테이블
- Desktop/Mobile 반응형 차이가 있을 때만 작성
- 차이가 없으면 "동일" 표기

### 테스트 케이스
- 사용자 동작 기준으로 작성
- 성공 조건만 명시 (실패 케이스는 필요시 추가)
- 구체적이고 검증 가능한 결과 기술

### 프로토타입 링크
- Figma 프레임 또는 Published Site 링크
- 해당 기능을 확인할 수 있는 화면 직접 링크

## 예시

```markdown
### LAND-001 Header

**주요 기능**
고정 헤더 + CTA 버튼

**레이아웃**
| 요소 | Desktop | Mobile |
|------|---------|--------|
| 로고 | 좌측 | 좌측 |
| CTA | 우측 → /test | 우측 (작은 버튼) |
| 동작 | Fixed, 스크롤 시 배경 흰색 + 그림자 | 동일 |

**프로토타입**: [Header Frame](링크)

**테스트 케이스**
- "내 수면 상태 확인하기" 버튼 클릭
  - 성공: /test 페이지로 이동
- 페이지 스크롤
  - 성공: 헤더 배경 흰색 + 그림자 적용
```
